# Configure Providers

Dhara supports multiple LLM providers through three adapter layers:

1. **Built-in providers** — OpenAI and Anthropic (zero extra dependencies)
2. **pi-ai adapter** — 20+ providers via `@earendil-works/pi-ai`
3. **Custom providers** — Implement the `Provider` interface for any API

## Built-in Providers

### OpenAI

```bash
export OPENAI_API_KEY="sk-..."
dhara --provider openai --model gpt-4o
```

Works with any OpenAI-compatible endpoint:

```bash
dhara --provider openai --model your-model --base-url https://api.custom.ai/v1
```

### Anthropic

```bash
export ANTHROPIC_API_KEY="sk-ant-..."
dhara --provider anthropic --model claude-sonnet-4-20250514
```

## Setting a Default Provider

Set your default provider in the global config:

```bash
# Edit global config
nano ~/.dhara/config.json
```

```json
{
  "provider": "openai",
  "model": "gpt-4o"
}
```

Or set it per-project in `.dhara/settings.json`:

```json
{
  "provider": "google",
  "model": "gemini-2.5-flash"
}
```

## Streaming Support

Both built-in providers support SSE streaming. When enabled, you'll see tokens appear progressively:

```bash
# Streaming is automatic — no extra flags needed
dhara --provider openai --model gpt-4o
```

In JSON mode (`--json`), streaming events are emitted as `message:delta` objects.

## Custom Provider

Implement the `Provider` interface for any API:

```typescript
import type { Provider, CompleteParams, CompleteResult } from "@zosmaai/dhara/core";

class MyProvider implements Provider {
  async complete(params: CompleteParams): Promise<CompleteResult> {
    // Call your API here
    // Optionally stream via params.eventBus
    return {
      content: [{ type: "text", text: "Response from my provider" }],
      usage: { inputTokens: 100, outputTokens: 50 },
    };
  }
}
```
